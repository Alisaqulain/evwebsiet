# Vercel Deployment Guide

## Project Configuration

This Next.js project is configured for Vercel deployment with zero-configuration support.

### Build Settings (Auto-detected by Vercel)
- **Framework Preset**: Next.js
- **Build Command**: `next build` (auto-detected)
- **Output Directory**: `.next` (auto-detected)
- **Install Command**: `npm install` (auto-detected)

### Important Files
- `package.json` - Contains build script: `"build": "next build"`
- `next.config.js` - Next.js configuration optimized for Vercel
- `vercel.json` - Security headers and caching (optional)
- `app/sitemap.ts` - Auto-generates `/sitemap.xml` (Next.js handles this automatically)

### Vercel Settings
If deploying manually, use these settings:
- **Framework Preset**: Next.js
- **Root Directory**: `./` (root)
- **Build Command**: Leave empty (auto-detected)
- **Output Directory**: Leave empty (auto-detected)
- **Install Command**: Leave empty (auto-detected)

### Environment Variables (if needed)
Add these in Vercel Dashboard → Settings → Environment Variables:
- `RESEND_API_KEY` (if using email functionality)

### Deployment Steps
1. Push code to GitHub/GitLab/Bitbucket
2. Import project in Vercel
3. Vercel will auto-detect Next.js
4. Deploy!

### Troubleshooting
- ✅ Build script is correct: `"build": "next build"`
- ✅ No custom output directory needed (Next.js handles this)
- ✅ Sitemap is auto-generated by Next.js from `app/sitemap.ts`
- ✅ All static files are in `public/` directory
